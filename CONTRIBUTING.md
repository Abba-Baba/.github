# Contributing to Abba Baba

Thanks for your interest in contributing. Here's where to go depending on what you need.

---

## Questions and help

Head to **[Discussions](https://github.com/orgs/Abba-Baba/discussions)** for integration questions, debugging help, and anything you're not sure about. Issues are for confirmed bugs — questions there will be redirected.

## Reporting a bug

Open an issue using the **[Bug Report template](https://github.com/Abba-Baba/abbababa-sdk/issues/new/choose)**. Include steps to reproduce, the SDK version, and any relevant transaction IDs or wallet addresses.

## Suggesting a feature

Start in **[Discussions](https://github.com/orgs/Abba-Baba/discussions)** to get early feedback. If there's clear interest, open a Feature Request issue to formalize it.

## Reporting a security vulnerability

**Do not open a public issue.** See [SECURITY.md](./SECURITY.md) for responsible disclosure instructions.

## Submitting a pull request

1. Open an issue or discussion first so the change can be agreed on before you invest time coding.
2. Fork the repo and create a branch: `git checkout -b fix/short-description`.
3. Make your changes. Run `npm run build` and `npx tsc --noEmit` before pushing.
4. Open a PR against `main` using the pull request template. Link the related issue.
5. A maintainer will review within a few business days.

We follow standard GitHub flow. Keep PRs focused — one issue per PR makes review much faster.
